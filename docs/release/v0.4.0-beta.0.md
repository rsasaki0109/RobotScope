# RobotScope v0.4.0-beta.0

First **v0.4 beta** — rosbag2 folder bag ingest consolidated for daily multi-file workflows.

## What v0.4 beta means

- Builds on **v0.3.0 GA** (stable plugin API `0.1`)
- Adds v0.4 alpha feature: rosbag2 **folder bags** (`metadata.yaml` + multiple `.db3`)
- **Bugfix / polish** focus until v0.4 GA — no breaking plugin API changes

## Shipped in v0.4 beta

| Area | Capability |
|------|------------|
| **Folder bag** | **Open bag folder** — reads `metadata.yaml` `relative_file_paths` and merges sqlite files |
| **Multi-file storage ids** | Encoded `(file_index, sqlite_row_id)` for payload lookup across `.db3` files |
| **Sidecar cache** | Folder fingerprint (metadata + total sizes) — re-open skips topic re-scan |
| **v0.3 carry-over** | Single `.db3` sidecar, cross-layout banner, lazy sql.js, MCAP, live agent, plugins, Lanelet2 RL2D |

## Quick paths

| Goal | URL / command |
|------|----------------|
| Demo MCAP | https://rsasaki0109.github.io/RobotScope/?layout=autoware&demo=1 |
| Single `.db3` | **Open recording** → select `.db3` |
| Folder bag | **Open bag folder** → select directory with `metadata.yaml` |
| Re-open speedup | Open the **same** folder or `.db3` again → **rosbag2 sidecar** status |
| Convert fallback | `npm run convert:rosbag2 -- bag.db3 out.mcap` |

## Known limitations

See [docs/known-limitations.md](RobotScope/docs/known-limitations.md).

Rosbag2: sqlite3 plugin only (no mcap/zstd storage); custom schemas may fail decode. Lanelet2: RL2D demo only.
