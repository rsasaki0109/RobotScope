# RobotScope v0.1.0-alpha.0

**Open observability for ROS2 robots, Autoware, Nav2, MoveIt, and Physical AI — first alpha release.**

---

## Highlights

| Area | What you get |
|------|----------------|
| **Offline** | MCAP playback, TF tree, 3D scene, timeline scrubber, raw message decode |
| **Live** | WebSocket agent, follow timeline, record session to MCAP |
| **Sidecar** | Per-topic time index for faster reload (browser cache + CLI writer) |
| **Plugins** | Autoware, Nav2, MoveIt layout packs via manifest registry |
| **Agent** | Native ROS2 (rclpy) bridge with topic profiles |

RobotScope explains what the robot **sensed**, **believed**, **planned**, and **commanded** on one timeline — without replacing Foxglove, Rerun, or RViz; it targets ROS2-native semantics and an OSS, plugin-first wedge.

---

## Quick start

```bash
git clone https://github.com/your-org/RobotScope.git
cd RobotScope
npm install
npm run build
npm run dev
```

1. Drop `sample_data/demo-scene.mcap` (generate with `node scripts/create-tf-demo.mjs`) onto the viewer, **or**
2. Run `npm run demo:live-agent` and click **Connect Live** → `ws://127.0.0.1:8765`

### Layout demos

```bash
npm run demo:autoware   # ?layout=autoware
npm run demo:nav2       # ?layout=nav2
npm run demo:moveit     # ?layout=moveit
```

### ROS2 agent (Jazzy+)

```bash
source /opt/ros/jazzy/setup.bash
npm run demo:ros2-agent -- --profile autoware
```

See `RobotScope/agent/README.md` for profiles and recording notes.

## Sidecar index

**Browser:** After live recording, sidecar is stored in IndexedDB keyed by file fingerprint.

**CLI:**

```bash
npm run build
node scripts/write-sidecar.mjs path/to/recording.mcap
# → path/to/recording.mcap.robotscope/index.json
```

Convention: `{mcap}.robotscope/index.json` (see `docs/architecture.md`).

## Known issues & alpha caveats

- **Empty ROS graph:** Agent logs `No topics subscribed` until publishers appear — expected.
- **Recording start mid-session:** Sidecar TF counts reflect live buffer stats, not only recorded window; TF geometry is re-indexed from MCAP on reload.
- **Remote URL placeholder:** README clone URL uses `your-org/RobotScope` until the GitHub org is finalized.
- **No cloud:** Single-machine / browser workflow only in this alpha.

## Upgrade / install notes

- **Node.js** ≥ 20
- **ROS2** Jazzy (or adjust `setup.bash` path in `scripts/run-ros2-agent.sh`)
- Build core before CLI sidecar script: `npm run build`
