# RobotScope v0.3.0-beta.0

First **v0.3 beta** — rosbag2 sidecar cache consolidated for daily re-open workflows.

## What v0.3 beta means

- Builds on **v0.2.0 GA** (stable plugin API `0.1`)
- Adds v0.3 alpha feature: rosbag2 **IndexedDB sidecar** on topic re-scan skip
- **Bugfix / polish** focus until v0.3 GA — no breaking plugin API changes

## Shipped in v0.3 beta

| Area | Capability |
|------|------------|
| **Rosbag2 sidecar** | IndexedDB cache with `recording_source: rosbag2`; re-open `.db3` skips topic timestamp re-scan |
| **Sidecar format** | `[time_ns, storage_id]` tuples (distinct from MCAP `[time_ns, sequence?]`) |
| **Fast path** | `loadRosbag2Payloads()` + `indexRosbag2TfOnly()` — payloads + TF still loaded from sqlite |
| **v0.2 carry-over** | Cross-layout recipe banner, lazy sql.js rosbag2 ingest, MCAP, live agent, plugins, Lanelet2 RL2D |

## Quick paths

| Goal | URL / command |
|------|----------------|
| Demo MCAP | https://rsasaki0109.github.io/RobotScope/?layout=autoware&demo=1 |
| Open rosbag2 | **Open recording** → select `.db3` (first open builds sidecar) |
| Re-open speedup | Open the **same** `.db3` again → status shows **rosbag2 sidecar** |
| Convert fallback | `npm run convert:rosbag2 -- bag.db3 out.mcap` |

## Known limitations

See [docs/known-limitations.md](RobotScope/docs/known-limitations.md).

Rosbag2: single `.db3`, payloads always reloaded, no folder bags yet. Lanelet2: RL2D demo only.
