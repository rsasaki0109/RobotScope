# RobotScope v0.2.0-alpha.1

**Rosbag2 SQLite** browser ingest — open `.db3` recordings without MCAP conversion.

## Highlights

- **`openRosbag2()`** — sql.js indexes standard rosbag2 `topics` + `messages` tables
- Viewer **Open recording** accepts `.db3` / `.sqlite3`
- Same query API as MCAP (TF tree, timeline, plugin snapshots, failure recipes when topics match)
- Fallback: `npm run convert:rosbag2 -- input.db3 output.mcap`
