# RobotScope v1.1.0-beta.0

First **v1.1 beta** — service call gateway consolidated for local live Trigger workflows.

## What v1.1 beta means

- Builds on **v1.0.0 GA** (stable plugin API `0.1`)
- Adds v1.1 alpha feature: **service call gateway** (`std_srvs/srv/Trigger` on allowlisted services)
- **Bugfix / polish** focus until v1.1 GA — no breaking plugin API changes

## Shipped in v1.1 beta

| Area | Capability |
|------|------------|
| **Service gateway** | Agent **`--allow-service`** + viewer **Allow service calls** + **Call trigger** |
| **Live protocol** | `command.service_call` + `command.service_result` |
| **Command gateway** | v1.0 carry-over — 6-DOF Twist editor + publish opt-in |
| **Read-only playback** | MCAP / rosbag2 / GitHub Pages demo unchanged |
| **v1.0 carry-over** | Command gateway, OSM maps, rosbag2 folder bags, failure recipes, plugins |

## Quick paths

| Goal | URL / command |
|------|----------------|
| Demo MCAP | https://rsasaki0109.github.io/RobotScope/?layout=autoware&demo=1 |
| Demo live + service | `npm run demo:live-agent` → Connect Live → Allow service calls → Call trigger |
| Native ROS | `npm run demo:ros2-agent -- --profile autoware --allow-service /robotscope/demo/trigger` |
| Folder bag | **Open bag folder** → `metadata.yaml` + `.db3` |

## Known limitations

See [docs/known-limitations.md](RobotScope/docs/known-limitations.md).

Trigger shortcut only; no action gateway; plugin manifest permissions not enforced at runtime.
