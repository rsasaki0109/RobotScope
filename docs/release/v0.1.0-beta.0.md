# RobotScope v0.1.0-beta.0

First **beta** milestone — scope frozen for v0.1, documentation consolidated, alpha feature set complete.

## What beta means

- **API / plugin manifest** may still change before v1.0, but core workflows are stable enough for daily Autoware/Nav2/MoveIt debug demos.
- Alpha releases (`0.1.0-alpha.0` … `alpha.8`) added failure recipes, GitHub Pages, unified timeline, and live recipe evaluation.
- Beta focuses on **clarity** (docs, limitations) rather than large new subsystems.

## Shipped in v0.1 beta

| Area | Capability |
|------|------------|
| **Ingest** | MCAP playback, sidecar index, live WebSocket agent, Record Live → MCAP |
| **3D** | TF tree, trajectories, occupancy mesh, lanelet centerlines |
| **Plugins** | Autoware, Nav2, MoveIt layouts + topic profiles |
| **Failure recipes** | 7 recipes across stacks; panel highlight + unified timeline strip |
| **Deploy** | GitHub Pages static demo (`?demo=1`, `?live=1`) |
| **Agent** | Python ROS2 bridge + MCAP replay demo agent |

## Quick paths

| Goal | Command / URL |
|------|----------------|
| Online demo | https://rsasaki0109.github.io/RobotScope/?layout=autoware&demo=1 |
| Local dev | `npm install && npm run dev` |
| Live replay | `npm run demo:live-agent` + Connect Live |
| All-stack recipes | Scrub demo MCAP — timeline footer (any layout) |

## Known limitations

See [docs/known-limitations.md](RobotScope/docs/known-limitations.md).

High level: no rosbag2-native reader, no Lanelet2 full mesh, read-only (no cmd gateway), heuristic failure recipes.
