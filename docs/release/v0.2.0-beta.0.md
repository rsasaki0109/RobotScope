# RobotScope v0.2.0-beta.0

First **v0.2 beta** — alpha feature set consolidated; rosbag2 ingest scoped for daily use.

## What v0.2 beta means

- Builds on **v0.1.0 GA** (stable plugin API `0.1`)
- Adds v0.2 alpha features: cross-layout recipe banner + rosbag2 SQLite ingest
- **Bugfix / polish** focus until v0.2 GA — no breaking plugin API changes

## Shipped in v0.2 beta

| Area | Capability |
|------|------------|
| **Failure recipes** | Cross-layout banner — click chip → switch Autoware/Nav2/MoveIt layout |
| **Rosbag2** | Browser `.db3` playback (sql.js); `npm run convert:rosbag2` fallback |
| **Bundle** | sql.js loaded **on demand** when opening rosbag2 (demo MCAP path stays lean) |
| **v0.1 carry-over** | MCAP, live agent, plugins, Lanelet2 RL2D, GitHub Pages demo |

## Quick paths

| Goal | URL / command |
|------|----------------|
| Demo MCAP | https://rsasaki0109.github.io/RobotScope/?layout=autoware&demo=1 |
| Open rosbag2 | **Open recording** → select `.db3` |
| Convert fallback | `npm run convert:rosbag2 -- bag.db3 out.mcap` |

## Known limitations

See [docs/known-limitations.md](RobotScope/docs/known-limitations.md).

Rosbag2: single `.db3`, common CDR types, no sidecar cache yet. Lanelet2: RL2D demo only.
