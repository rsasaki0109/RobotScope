# RobotScope v0.7.0-alpha.0

First **v0.7 alpha** — OSM **regulatory elements** (traffic rules geometry).

## Highlights

- **`parseLaneletOsm()`** now reads `<relation type=regulatory_element>` with `subtype` tags
- Resolves geometry members: `refers`, `ref_line`, `crosswalk_polygon`, `light_bulbs`
- Map panel shows regulatory count + subtype breakdown (e.g. `traffic_light×3`)
- 3D + 2D preview overlays for stop lines and rule polygons
- v0.6 lanelet relations unchanged; native boost bin still unknown

## Try it

1. Open a recording
2. **Load map OSM** → Autoware `.osm` export with regulatory relations
3. Autoware layout → Map panel shows regulatory element count + preview

Maps without regulatory relations behave like v0.6.
