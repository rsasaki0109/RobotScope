# RobotScope v0.5.0-beta.0

First **v0.5 beta** — Lanelet2 OSM sidecar consolidated for daily map overlay workflows.

## What v0.5 beta means

- Builds on **v0.4.0 GA** (stable plugin API `0.1`)
- Adds v0.5 alpha feature: **Load map OSM** sidecar when native `LaneletMapBin` is unknown
- **Bugfix / polish** focus until v0.5 GA — no breaking plugin API changes

## Shipped in v0.5 beta

| Area | Capability |
|------|------------|
| **OSM sidecar** | Parse Autoware `.osm` (`local_x`/`local_y` + ways) — 2D Map panel + 3D polylines |
| **Native bin gap** | `LaneletMapBin` boost serialization still unknown — OSM complements RL2D demo MCAP |
| **v0.4 carry-over** | Rosbag2 folder bags + sidecar, cross-layout banner, MCAP, live agent, plugins |

## Quick paths

| Goal | URL / command |
|------|----------------|
| Demo MCAP | https://rsasaki0109.github.io/RobotScope/?layout=autoware&demo=1 |
| Load OSM | **Load map OSM** → `.osm` export alongside recording |
| Folder bag | **Open bag folder** → `metadata.yaml` + `.db3` |
| README hero regen | `npm run capture:readme-hero` |

## Known limitations

See [docs/known-limitations.md](RobotScope/docs/known-limitations.md).

Lanelet2: OSM ways only (no relations); native boost bin not parsed. Read-only — no command gateway.
