# RobotScope v0.8.0-beta.0

First **v0.8 beta** — permission-gated command gateway consolidated for local live debug workflows.

## What v0.8 beta means

- Builds on **v0.7.0 GA** (stable plugin API `0.1`)
- Adds v0.8 alpha feature: **command gateway** (`command.publish` with agent allowlist + viewer opt-in)
- **Bugfix / polish** focus until v0.8 GA — no breaking plugin API changes

## Shipped in v0.8 beta

| Area | Capability |
|------|------------|
| **Command gateway** | Agent **`--allow-publish TOPIC`** + viewer **Allow publish** + **Zero cmd_vel** |
| **Live protocol** | Session `capabilities.command_publish` + `command.publish_result` |
| **Read-only playback** | MCAP / rosbag2 / GitHub Pages demo unchanged — publish is live-only |
| **v0.7 carry-over** | OSM regulatory + lanelet relations, rosbag2 folder bags, failure recipes, plugins |

## Quick paths

| Goal | URL / command |
|------|----------------|
| Demo MCAP | https://rsasaki0109.github.io/RobotScope/?layout=autoware&demo=1 |
| Demo live + gateway | `npm run demo:live-agent` → Connect Live → Allow publish → Zero cmd_vel |
| Native ROS publish | `npm run demo:ros2-agent -- --profile autoware --allow-publish /cmd_vel` |
| Folder bag | **Open bag folder** → `metadata.yaml` + `.db3` |

## Known limitations

See [docs/known-limitations.md](RobotScope/docs/known-limitations.md).

Gateway: zero `/cmd_vel` only; no service/action gateway; plugin manifest permissions not enforced at runtime.
