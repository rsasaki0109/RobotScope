# RobotScope v1.2.0-beta.0

First **v1.2 beta** — action goal gateway consolidated for local live Fibonacci workflows.

## What v1.2 beta means

- Builds on **v1.1.0 GA** (stable plugin API `0.1`)
- Adds v1.2 alpha feature: **action goal gateway** (`example_interfaces/action/Fibonacci` on allowlisted actions)
- **Bugfix / polish** focus until v1.2 GA — no breaking plugin API changes

## Shipped in v1.2 beta

| Area | Capability |
|------|------------|
| **Action gateway** | Agent **`--allow-action`** + viewer **Allow action goals** + **Send Fibonacci** |
| **Live protocol** | `command.action_send_goal` + `command.action_result` |
| **Service gateway** | v1.1 carry-over — Trigger service calls + opt-in |
| **Command gateway** | v1.0 carry-over — 6-DOF Twist editor + publish opt-in |
| **Read-only playback** | MCAP / rosbag2 / GitHub Pages demo unchanged |
| **v1.1 carry-over** | Service gateway, OSM maps, rosbag2 folder bags, failure recipes, plugins |

## Quick paths

| Goal | URL / command |
|------|----------------|
| Demo MCAP | https://rsasaki0109.github.io/RobotScope/?layout=autoware&demo=1 |
| Demo live + action | `npm run demo:live-agent` → Connect Live → Allow action goals → Send Fibonacci |
| Native ROS | `npm run demo:ros2-agent -- --profile autoware --allow-action /robotscope/demo/fibonacci` |
| Folder bag | **Open bag folder** → `metadata.yaml` + `.db3` |

## Known limitations

See [docs/known-limitations.md](RobotScope/docs/known-limitations.md).

Fibonacci goal send only; no feedback/result UI; plugin manifest permissions not enforced at runtime.
