# RobotScope v1.7.0-beta.0

First **v1.7 beta** — Lanelet2 Boost binary parsing consolidated for daily map workflows.

## What v1.7 beta means

- Builds on **v1.6.0 GA** (stable plugin API `0.1`)
- Adds v1.7 alpha feature: **Boost `LaneletMapBin.data`** heuristic parse + 3D bounds
- **Bugfix / polish** focus until v1.7 GA — no breaking plugin API changes

## Shipped in v1.7 beta

| Area | Capability |
|------|------------|
| **Boost bin parse** | `parseBoostLaneletMap()` — archive detect + boundary polygons + Map panel **(Boost bin)** |
| **OSM sidecar** | v0.7 carry-over — ways + lanelet relations + regulatory geometry |
| **RL2D demo** | GitHub Pages demo MCAP vector map unchanged |
| **Action stack** | v1.6 carry-over — preempt · timeline · cancel · tracking · gateways |
| **Read-only playback** | MCAP / rosbag2 / GitHub Pages demo unchanged |

## Known limitations

See [docs/known-limitations.md](RobotScope/docs/known-limitations.md).

Boost parsing is **heuristic alpha scope** — full lanelet topology, regulatory refs, and cross-Boost portability not guaranteed. Use [lanelet2-osm.md](RobotScope/docs/lanelet2-osm.md) sidecar for complex maps.
