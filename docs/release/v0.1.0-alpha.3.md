# RobotScope v0.1.0-alpha.3

Autoware debug UX update: **Perception Objects panel** + **phantom obstacle stop** failure recipe demo.

## Highlights

- **Perception Objects panel** — tracks `/perception/object_recognition/objects` with confidence stats and brief-spike detection
- **Phantom obstacle stop recipe** — when planning stalls on a short trajectory and a low-confidence object appears briefly, highlights Planning + Perception panels
- Demo MCAP encodes the phantom-stop scenario from **~1.4s** onward (stalled plan + brief false positive)

## Try it

```bash
npm install && npm run build
node scripts/create-tf-demo.mjs
npm run demo:autoware
```

Scrub the timeline:

- **~1.0–1.3s** — normal driving; localization drift may appear late in the clip
- **~1.4s+** — **Phantom obstacle stop suspected** recipe activates (planning length collapses + perception spike)
